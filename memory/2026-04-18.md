Built a full state-machine orchestrator for guiding open-source models through attacks:
- **`scripts/orchestrator.sh`** — main entry (init/think/report/error/status/decide)
- **`scripts/orch-think.py`** — decision engine, outputs exact commands per phase (13KB)
- **`scripts/orch-report.py`** — smart result parser, auto-transitions between phases
- **`scripts/orch-error.py`** — error handler with fix suggestions and retry tracking
- **`scripts/orch-status.py`** — state viewer
- **`knowledge-base/error-recovery.md`** — comprehensive error recovery playbook (6KB)
- Updated `AUTOPILOT.md` to wire orchestrator as primary attack mode

**The loop for dumb models:**
1. `bash scripts/orchestrator.sh init TARGET`
2. `bash scripts/orchestrator.sh think` → get exact command
3. Run the command
4. `bash scripts/orchestrator.sh report "result"` → auto-transitions to next phase
5. GOTO 2

Phases: recon → web_enum → exploit → postex → report → complete
State tracks: ports, services, hostnames, creds, shells, flags, findings, errors
